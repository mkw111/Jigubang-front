# 프로젝트 이슈 해결 로그

## 1. 의존성 설치 및 구동 실패 이슈 (2026-05-18)

### 문제 상황
- `npm start` 실행 시 `'react-scripts'은(는) 내부 또는 외부 명령... 아닙니다.` 에러 발생.
- `npm install` 실행 시 48개의 패키지만 감사(audit)하고 매우 빠르게 종료됨. (실제 의존성 설치 안 됨)

### 원인 분석
- **버전 충돌**: `package.json`에 정의된 `react` 버전(v19)과 `react-scripts`(v5.0.1) 간의 peer dependency 충돌로 추정됨.
- `npm` 최신 버전(v11)은 의존성 충돌에 엄격하여, 충돌 발생 시 설치를 중단하거나 최소한의 패키지만 설치하는 경향이 있음.

### 해결 방법
1. 기존에 잘못 생성된 파일 삭제: `node_modules`, `package-lock.json`
2. 의존성 강제 설치: `npm install --legacy-peer-deps` 실행
   - 이 옵션은 버전 충돌을 무시하고 설치를 강행함.

## 2. SMS 인증 기능 수정 (완료 - 2026-05-18)

### 목표
- `register_guide.pdf` 8페이지 디자인을 반영하고, 실제 서버 API와 연동하여 SMS 인증 기능을 정상화함.

### 변경 사항
- **UI 구현**:
    - `CertPage.tsx` 및 `CertPage.css` 전면 수정.
    - 이름 입력, 휴대폰 번호 입력, 인증번호 입력 필드 구현.
    - **3분 타이머(03:00)** 기능 추가 (남은 시간 표시).
    - 인증 성공 시 '다음' 버튼 활성화 로직 적용.
- **API 연동**:
    - `axios`를 사용하여 `Jigubang-back` 서버와 통신.
    - 인증번호 요청: `POST /api/auth/hp-cert/send`
    - 인증번호 확인: `POST /api/auth/hp-cert/confirm`
- **보안/편의성**:
    - 인증 완료 후 입력 필드 `disabled` 처리하여 데이터 무결성 보장.
    - 'X' 버튼을 통한 이름 입력 필드 초기화 기능.

### 결과
- 이제 실제 휴대폰 번호를 입력하고 인증번호를 받아 가입 절차를 진행할 수 있는 상태임.
